# Book Search App

A semantic book search engine powered by sentence embeddings and FAISS, with a Streamlit web interface. Search through **16,500+ book summaries** using natural language queries.

---

## Demo

Type a query like *"a story about war and redemption"* or *"dystopian society controlled by the government"* and get the 5 most semantically relevant books in return.

---

## Project Structure

```
├── booksummaries.txt       # Raw dataset (CMU Book Summary Dataset)
├── book.csv                # Processed CSV (generated by the notebook)
├── book_embeddings.npy     # Precomputed embeddings (generated by precompute_embeddings.py)
├── code.ipynb              # Exploration notebook: data prep, embedding prototype & search tests
├── precompute_embeddings.py # Script to generate and save embeddings
└── app.py                  # Streamlit web application
```

---

## 🧠 How It Works

1. **Data preparation** — `booksummaries.txt` is parsed and converted to a structured CSV (`book.csv`) with columns: `Title` and `Summary`.
2. **Exploration & prototyping** — `code.ipynb` covers the full pipeline: data preparation, embedding generation, FAISS index construction and semantic search tests. It serves as the experimental ground that led to the final app.
3. **Embedding generation** — `precompute_embeddings.py` is the standalone script extracted from the notebook to generate and save all vectors to `book_embeddings.npy`.
4. **Indexing** — All vectors are indexed with FAISS (`IndexFlatIP`) for fast inner-product similarity search.
5. **Search** — At query time, the user's input is encoded with the same model and the 5 nearest neighbors are retrieved from the index.

## 🚀 Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/your-username/book-search-app.git
cd book-search-app
```

### 2. Install dependencies

```bash
pip install streamlit pandas numpy faiss-cpu sentence-transformers
```

### 3. Prepare the data

Run the first cell of `code.ipynb` to convert `booksummaries.txt` into `book.csv`. You can also explore the full pipeline (embeddings, FAISS index, search tests) directly in the notebook before running the app.

```bash
jupyter notebook code.ipynb
```

### 4. Precompute embeddings

This step encodes all 16,500+ books and saves the vectors to disk. It takes ~13 minutes on CPU.

```bash
python precompute_embeddings.py
```

This will generate `book_embeddings.npy`.

### 5. Launch the app

```bash
streamlit run app.py
```

Then open [http://localhost:8501](http://localhost:8501) in your browser.

## 📊 Dataset

This project uses the **CMU Book Summary Dataset**, which contains plot summaries for 16,559 books extracted from Wikipedia, along with metadata (author, publication date, genres).

> Bamman, D. & Smith, N. (2013). *New Alignment Methods for Discriminative Book Summarization*. Presented at the ACL Workshop on Natural Language Processing for Digital Humanities.
